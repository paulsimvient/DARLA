# DARLA Studio UI Standards

## UI goal

DARLA should feel like a professional mission engineering tool:
- ArcGIS Pro
- Unreal Editor
- VS Code
- Simulink
- AFSIM

Not like a card dashboard.

## Layout rules

1. Map/timeline/explorer stay in Mission.
2. Graphs should explain, not overwhelm.
3. Default views show critical paths only.
4. Inspectors should never be empty.
5. Every panel must respond to shared selection.
6. Evidence should be one click away.
7. Raw JSON belongs in advanced/debug views only.

## Causal graph rules

- Never show all edges by default.
- Highlight one critical path.
- Ghost non-critical links.
- Labels appear on hover/selection.
- Inspector explains evidence and counterfactuals.

## Decision UI rules

- COAs use a compact matrix.
- One recommended action at top.
- Gates are columns, not stacked cards.
- Evidence opens in drill-down.

## Text standards

Use careful claims:
- "supports"
- "indicates"
- "within the simulation envelope"
- "requires review"

Avoid:
- "proves"
- "guarantees"
- "autonomously determines truth"
